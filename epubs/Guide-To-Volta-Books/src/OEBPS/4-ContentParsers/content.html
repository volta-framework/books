Content Parsers

A DocumentNode can contain content in different formats, such as plain text or HTML. Each type of content requires a corresponding content parser, which is a PHP class implementing the Volta\Component\Books\ContentParserInterface.

To specify the content-type of a DocumentNode, just provide the content file with the correct extension. Volta Books will search for a registered ContentParser for this extension in the current configuration. If none is found, an error will occur. You can create your own content parsers for specific extensions, which we'll discuss further. Volta Books includes the following built-in ContentParsers:

  • Volta\Component\Books\ContentParsers\TxtParser for content.txt files
  • Volta\Component\Books\ContentParsers\HtmlParser for content.html and content.htm files
  • Volta\Component\Books\ContentParsers\PhpParser for content.php and content.phtml files
  • Volta\Component\Books\ContentParsers\XhtmlParser for content.xhtml files
  • Volta\Component\Books\ContentParsers\TxtParser for content.md files
