Glossary

BookNode
When a DocumentNode has no parent directory classified as a DocumentNode, it is considered a BookNode.
DocumentNode
To earn the classification of a DocumentNode, specific criteria must be met. A directory must contain two obligatory files within it. The first file is named content.* and serves as the repository for the document's content. The second file, meta.json is essential, and it must adhere to the standards of a valid JSON file, serving as the storage space for crucial metadata.
ResourceNode
Any files found within the DocumentNodes directory, excluding those named content.* or meta.json, are considered ResourceNodes.
RootNode
See BookNode
Volta Application
A Volta Component that enables collaboration between various components.
Volta Book Format
The format defined by Volta-Component-Books used for storing book data.
Volta Component
A PHP package that can be installed via Composer, conforming to the specific file and directory structure outlined by the Volta Framework.
Volta Framework
An collection of PHP packages, each adhering to a uniform file and directory structure, used to construct a complete Volta Application instance.
